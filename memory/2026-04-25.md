### 关键规则
- 严禁执行删除操作（deleteCard / deleteTask 等）
- 修改 dashboard card 前必须先备份原内容
- 登录接口：`POST https://ones.cn/api/project/auth/login`
- 后续请求必须带：`Ones-User-Id` + `Ones-Auth-Token`
- GraphQL 入口：`POST /api/project/team/{TEAM}/items/graphql`
- 创建任务接口：`POST /api/project/team/{TEAM}/tasks/add`
- 创建任务时必须同时提供：`issue_type_uuid` + `issue_type_scope_uuid`
- `issue_type_scope_uuid` 需从目标项目现有任务中查询，不能想当然复用
- dashboard card 无法可靠通过 API 创建，通常需要用户先在 UI 手动创建 announcement 卡片，再由 agent 更新内容

### 已知限制
- 任务更新（改状态/负责人）接口当前不稳定：PUT/PATCH 404，POST `/tasks/update` 500
- card 创建 API `addCard` 返回 500
- GraphQL 某些字段不可用：`description_text`、`create_time`、`discussion_count`
