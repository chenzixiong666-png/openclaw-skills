## 优先级判断标准

### P0（高优先级）
- 核心功能路径
- 阻断性问题
- 影响主要业务流程
- 安全性相关问题

### P1（中优先级）
- 重要但非核心功能
- 影响用户体验
- 边界条件处理
- 状态显示逻辑

### P2（低优先级）
- 优化类功能
- 视觉细节
- 极端边界情况
- 辅助功能

## 常见交互模式测试点

### 首次使用流程
1. 入口 new 标签显示
2. 首次点击触发引导
3. 引导内容展示
4. 引导关闭逻辑
5. 引导后 new 标签消失

### 购买流程
1. 未拥有状态显示购买按钮
2. 已拥有状态显示标识
3. 点击购买进入支付
4. 支付成功状态更新
5. 支付失败处理

### 扫码流程
1. 扫码入口显示
2. 打开扫码界面
3. 扫描识别
4. 解锁确认
5. 同步验证

### 状态同步
1. 本地状态显示
2. 服务器同步
3. 多端一致性
4. 刷新后状态
