# 周报输出模板

用于 ONES 项目周报汇总输出时的固定结构。

## 标题

`{项目名}单品项目测试周报（{起始日期YYYYMMDD}~{结束日期MMDD}）`

## 主体结构

### 1、{起始日期YYYYMMDD}~{结束日期MMDD}

#### 1）项目基本信息
- 计划测试内容
- 测试代表
- 固件版本
- 本周计划人力投入
- APP版本
- 实际测试人力投入
- 本轮提测时间
- 其他投入时间
- 目标完成时间
- 其他投入时间说明
- 测试范围
- 当前轮次

#### 2）进展与缺陷

##### 测试进展
- 当前轮次测试进度
- 本周完成情况
- 缺陷新增数量
- ONES 测试计划 / 项目说明

##### 阻塞情况
- 默认留“待补充”
- 不要擅自编造详细阻塞项

##### 严重缺陷
- 优先列出 严重 / 较高 缺陷
- 每条格式：`项目编号【类型】标题 — 优先级 — 状态`
- 必要时补负责人

##### 缺陷看板
- 缺陷总数
- 各缺陷类别数量
- 各类别状态分布
- 各类别优先级分布
- 当用户希望直接把 ONES 缺陷汇总并入这里时，优先使用精简模块汇总：先写类别数量，再按模块分组；每个模块补一句汇总描述；缺陷条目优先使用 `缺陷简称【状态】【优先级】[【跳转】](链接)`，避免裸贴长链接

#### 3）风险与需求
- 概括缺陷密集区域
- 概括尚未进入修复闭环的问题
- 点出高优先级需求缺陷或阻塞风险

#### 4）下周测试计划
- 跟进缺陷整改
- 跟进回归
- 补充阻塞项/截图/正式版本信息

## 项目基本信息表格默认值规则

未知字段统一填：
- 版本类：`待确认`
- 日期类：`待确认`
- 人力类：`待确认`
- 说明类：`/`
- 当前轮次：`待确认`

常用可由调用方传入的字段：
- `report_start`
- `report_end`
- `plan_test_content`
- `test_owner`
- `firmware_version`
- `app_version`
- `planned_manpower`
- `actual_manpower`
- `submit_time`
- `other_time`
- `target_finish_time`
- `other_time_note`
- `test_scope`
- `current_round`

## 输出原则

- 未知字段统一写“待确认”或“待补充”
- 不要伪造版本号、日期、负责人等关键字段
- 用户提到“按昨天周报格式”时，优先使用这个结构
- 如果用户要求把缺陷汇总直接作用到“缺陷看板”，默认采用更短的模块化汇总版本，而不是长表格或裸 URL 列表
